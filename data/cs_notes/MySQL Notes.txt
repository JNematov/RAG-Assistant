SQL: Tables with fixed rows and columns, ex: Oracles, MySQL, Microsoft SQL Server and PostgreSQL
NoSQL: Document: JSON documents, Key-value: key-value pairs, Wide-column: tables with rows and dynamic columns, Graph: nodes. Document: MongoDB and CouchDB, Key-value: Redis and DynamoDB, Wide-column: Cassandra and HBase, Graph: Neo4j and Amazon Neptune. 


SQL - MySQL: Database → Tables → Rows → Columns
NoSQL Example - MongoDB: Database → Collections → Documents → Fields


- primary key constant basically uniquely identifies each record in a table, it must contain unique values and a table can only have a single primary key


-foreign key is related to primary key, a foreign key is a field in one table referring to the primary key in another table

MYSQL COMMANDS (the ones you actually type every day)
====================================================
1. Connection
mysql -u root -p
USE database_name;
SHOW DATABASES;
SHOW TABLES;
DESC table_name;
2. Database
CREATE DATABASE mydb CHARACTER SET utf8mb4;
DROP DATABASE mydb;
3. Table DDL
CREATE TABLE users (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) UNIQUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
DROP TABLE users;
TRUNCATE TABLE users;
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
ALTER TABLE users DROP COLUMN phone;
4. Data DML
INSERT INTO users (name, email) VALUES ('John', 'j@x.com');
INSERT INTO users (name) VALUES ('A'), ('B');
SELECT * FROM users WHERE age > 25 ORDER BY age DESC LIMIT 10;
SELECT COUNT(*) FROM users;
UPDATE users SET age = age + 1 WHERE id = 5;
DELETE FROM users WHERE id = 10;
5. JOINs
SELECT o.id, u.name FROM orders o
INNER JOIN users u ON o.user_id = u.id;
LEFT JOIN users u ON o.user_id = u.id;
6. Aggregation
SELECT country, COUNT(*), AVG(age) FROM users GROUP BY country;
SELECT DATE(created_at) AS day, COUNT(*) FROM orders
GROUP BY day HAVING COUNT(*) > 100;
7. Indexes
CREATE INDEX idx_email ON users(email);
CREATE UNIQUE INDEX idx_uniq_email ON users(email);
SHOW INDEX FROM users;
DROP INDEX idx_email ON users;
8. Users & Privileges
CREATE USER 'app'@'%' IDENTIFIED BY 'strong-pass';
GRANT ALL ON mydb.* TO 'app'@'%';
GRANT SELECT, INSERT, UPDATE ON mydb.* TO 'reader'@'%';
SHOW GRANTS FOR 'app'@'%';
FLUSH PRIVILEGES;
9. Handy
SELECT VERSION();
SELECT NOW();
SOURCE /path/to/dump.sql;
mysqldump -u root -p mydb > backup.sql   (from terminal)
mysql -u root -p mydb < backup.sql       (import)
